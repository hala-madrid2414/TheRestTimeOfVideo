# B站视频剩余时长计算器文档与构建脚本优化设计

## 1. 背景与目标

当前扩展的 `README.md` 将使用者和开发者的安装步骤分离开来，导致使用者可能不清楚需要通过 `npm install` 和 `npm run build` 来获取最新的构建产物（`dist/` 目录）。同时，用户在执行构建后，可能不知道如何将产物加载到浏览器中。

目标：
- 优化 `README.md` 结构，将通用的“从源码构建并安装扩展”的流程前置为统一的“快速开始”章节。
- 在执行构建命令（`npm run build` 或 `npm run verify`）成功后，在终端输出醒目的引导信息，指导用户如何在 Chrome 中加载扩展。

## 2. README.md 结构优化方案

文档结构将调整如下：

1. **标题与简介**（包含 Badges 和截图）
2. **快速开始 (Quick Start)** (新增/合并)
   - 环境要求 (Node.js)
   - 获取源码与安装依赖 (`npm install`)
   - 构建扩展 (`npm run build`)
   - 在浏览器中加载扩展 (Chrome/Edge 的具体步骤)
3. **功能特性与说明** (面向使用者)
   - 功能列表
   - 已知限制
   - 常见问题 (FAQ)
4. **开发者指南** (面向深度开发)
   - 技术栈与项目结构
   - 架构简述
   - 开发调试流程 (如何调试 Popup, Content Script, Background)
5. **隐私说明**
6. **开源协议**

## 3. 构建脚本引导输出方案

在现有的 `scripts/check-dist.mjs`（该脚本在 `npm run build` 和 `npm run verify` 中都会被调用或后续调用）的结尾处，添加一段高亮的终端输出。

利用终端控制字符（或直接输出清晰的文本结构）来提示用户：
```text
===================================================
✅ 构建成功！产物已生成至: dist/ 目录

🚀 如何在浏览器中安装此扩展：
1. 打开 Chrome，在地址栏输入 chrome://extensions/
2. 开启右上角的【开发者模式】
3. 点击左上角【加载已解压的扩展程序】
4. 选择本项目下的 dist 目录即可完成安装！
===================================================
```

这样用户在敲完 `npm run build` 后就能立刻看到下一步该怎么做。
